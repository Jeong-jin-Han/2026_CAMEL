## 📋 개요 (한 줄 흐름)

> [오프닝: **AI 인프라**에 기여하고 싶었다 → 세 전공이 그 목표 하나로 모임] → **1단계 (전력 공급 측면)**: AI 데이터센터 전력 수요 ↑ → SMR 같은 차세대 원전 → 원전 시뮬레이션 + 자율운전 목표 → 세 전공 (원자력 복전 = 시뮬레이션 / 수리 부전 = 수학 기반 / 전산 주전 = 자율운전) → 원자력 대학원 수업 MATLAB + MARS → **2단계 (시스템 측면으로)**: 자율운전 위해 전산 수업 → 시스템에 끌림 → **세 길 비교, 두 축: ① 배경지식의 양 ② 논문 영향력 지속성** · AI = 진입 빠르나 배경지식 적고 paradigm 빨리 바뀜 (NeRF→3DGS 1~2년) / 원자력 = 배경지식 크지만 규제로 산업 적용 느림 / **컴퓨터 시스템 = 여러 layer 누적 이해 필요 → 배경지식 ↑ → LLM 탐색 비용 ↑ → 사람 직관 가치 ↑ + 영향력 오래 가 한 흐름 깊이 이어가기 + 규제 없음** → AI 인프라 안에서 전력 공급에서 시스템으로 옮겨간 셈 → **3단계 (메모리 용량 한계과 그 풀이)**: 그래픽카드 가격 차이 → 마개조 GPU → "메모리 용량 한계 우회 가능" → 칩 붙이기는 물리 한계 → **하드웨어·소프트웨어 같이 설계** → TrainingCXL → 그 밑바탕엔 LightPC → **클로징**: LightPC가 한 대 안에서 풀어놨음 → 여러 대로 가면 새 부분들 더해짐 → **LightPC 방식을 여러 대 컴퓨터까지** + CAMEL Lab 호명 + "이번엔 진심"

## 🎤 풀 버전 (낭독 약 4분 45초)

> 굵게 표시된 부분이 **고유 이름** — 정확히 발음. 나머지는 일상어로 자연스럽게.

### (오프닝 · ~30초)

안녕하세요. 전산학부 주전공, 원자력및양자공학 복수전공, 수리과학 부전공을 하는 학부생 한정진입니다. 졸업은 2027년 2월 예정이고, 지금 원자력 전공 두 과목만 남았습니다.

저는 처음부터 **AI 인프라**에 기여하고 싶었습니다. 처음엔 **전력 공급** 쪽에, 나중엔 **시스템** 쪽에 관심이 갔고, 세 전공이 그 목표 하나로 모이게 된 이야기를 짧게 말씀드리겠습니다.

### (1단계 — 전력 공급 측면, 그리고 세 전공 · ~1분)

처음엔 AI 데이터센터의 전력 수요가 빠르게 늘고 있다는 점이 눈에 들어왔습니다. 그 전력원, 특히 **SMR** 같은 차세대 원전을 직접 다뤄보고 싶어서 **원전 시뮬레이션과 자율운전**을 목표로 잡았고, 원자력 시뮬레이션을 위해 **원자력**을 복수전공으로, 그 수학적 기반이 필요해서 **수리과학**을 부전공으로, 자율운전을 해보려고 **전산학부**를 주전공으로 정했습니다.

실제로 **원자력 대학원 수업**에서 원자로 안의 열과 유체 흐름을 푸는 코드를 **MATLAB**으로 처음부터 구현해보고, 산업 표준 코드인 **MARS**와 결과를 비교 검증해봤습니다.

### (2단계 — 세 길을 비교, 시스템 쪽으로 옮겨가기 · ~2분)

그리고 자율운전을 직접 해보려고 컴퓨터를 더 깊이 공부하기 시작했는데, **전산학부 수업을 들으면서** 점점 원전 자체보다 그걸 떠받치는 **컴퓨터 시스템 쪽**에 더 끌리게 됐습니다.

사실 그때 세 길을 두고 제 나름대로 비교해봤습니다. 아직 깊이 연구를 해본 입장은 아니라 표면적 인상에 가깝지만, 제 눈에 들어온 차이는 두 가지였습니다 — **한 논문에 contribution을 더하는 데 필요한 배경지식의 양**, 그리고 **그 논문의 영향력이 얼마나 오래 가는가**입니다.

**AI 연구**는 한 논문 안에 모델 설계가 거의 다 들어 있어서 진입이 빠른 대신, 배경지식이 상대적으로 적게 느껴졌고 paradigm도 빠르게 바뀌는 것 같았습니다 — NeRF에서 3DGS로 가는 데 1~2년 정도였으니까요. **원자력**은 배경지식이 큰 분야로 보였는데, 인허가 규제로 산업 적용까지 너무 오래 걸리는 것이 마음에 걸렸습니다 — 새 원자로 하나 만드는 데 십몇 년이라, 제가 만든 것이 돌아가는 걸 보기도 전에 정년퇴직할 것 같다고 느꼈습니다.

반면 **컴퓨터 시스템**은 한 논문을 구현하려고만 해도 하드웨어부터 OS, application까지 여러 layer의 누적된 이해가 필요해 보였습니다. **배경지식이 많을수록 LLM이 따라잡는 비용도 커지니까, 한 사람의 직관이 오래 가치를 유지할 영역이 아닐까** 싶었고, **한 논문의 영향력도 더 오래 가서 한 흐름을 깊이 이어갈 수 있을 것 같다**는 점도 마음에 들었습니다. 규제도 강하지 않아 떠올린 아이디어를 직접 시도해보고 **상대적으로** 빠르게 결과를 볼 수 있다고 생각했습니다.

결과적으로 **전력 공급 쪽**에 있던 관심이 **시스템 쪽**으로 옮겨가게 됐습니다.

### (3단계 — 메모리 용량 한계과 그 풀이 · ~1분 30초) ⭐

그렇게 컴퓨터 시스템을 공부하다 보니 **그래픽카드**에 관심이 생겼습니다. 카드들을 살펴보다가, 같은 칩에 비슷한 카드인데도 **그래픽카드 메모리** 용량에 따라 가격이 몇 배씩 차이가 난다는 점이 눈에 띄었습니다. "왜 그렇게까지 차이가 날까" 싶어서 찾아보니 — 결국 메모리가 크면 그만큼 더 큰 AI 모델을 돌릴 수 있고, 그것이 그래픽카드의 시장 가치를 정한다는 사실을 알게 됐습니다.

그러다 인상 깊은 사례를 봤습니다. 누군가 그래픽카드를 뜯어 메모리 칩을 직접 더 붙여, 원래 용량의 몇 배짜리 카드로 개조해 큰 모델을 돌려낸 사례였습니다. 그걸 보고 "메모리 용량 한계은 우회할 수 있는 것이구나" 하는 생각이 들었습니다.

다만 칩을 직접 붙이는 방식은 결국 물리적 한계가 있다 보니, 좀 더 근본적인 방법이 없을까 찾던 중에 **하드웨어와 소프트웨어를 처음부터 같이 설계하는 방향**을 알게 됐습니다. 그래픽카드 메모리만으로 부족할 때, 본체 바깥에 따로 떨어진 메모리까지 그래픽카드가 자연스럽게 갖다 쓰게 만드는 방식입니다.

그 가능성을 가장 인상 깊게 보여준 연구가 **TrainingCXL**이었습니다. 그래픽카드가 본체 바깥의 큰 메모리까지 직접 갖다 쓰게 만든 사례입니다. 그 밑바탕에는 **LightPC**가 있었습니다. 사용자가 따로 손대지 않아도 컴퓨터 전원이 꺼졌다 켜질 때 작업 상태가 그대로 돌아오도록 만든 연구입니다.

### (클로징 · ~1분)

**LightPC**는 그 일을 한 대 컴퓨터 안에서는 풀어놨습니다. 다만 요즘 큰 AI 모델은 여러 대 컴퓨터로 흩어져서 같이 학습하다 보니, 컴퓨터들 사이를 오가는 메시지를 어떻게 같이 잡을지, 한 대만 전원이 나가면 어떻게 할지 같은 새로운 부분들이 더해지는 것 같았습니다.

그래서 저는 LightPC가 한 대 안에서 보여준 그 방식을, 여러 대 컴퓨터까지 가져가는 방향을 해보고 싶습니다. **CAMEL Lab**이 바로 이렇게 하드웨어와 소프트웨어를 같이 설계해 실제로 만들고 검증하는 곳이라, 제가 가장 배우고 싶은 환경이라고 생각했습니다.

아직 배울 게 많은 학부생입니다. 이번 세미나에서 많은 것을 배워가고 싶습니다. 잘 부탁드리겠습니다. 감사합니다.

## 📐 v2 → v3 변경 사항 표

| 위치 | v2 (정확함, 무거움) | v3 (정확함 + 가벼움) | 이유 |
|---|---|---|---|
| **오프닝 + 1단계 + 2단계 dual-side framing** | (v2 기준) AI 인프라 → SMR → 원자력 → 자율운전 → 시스템 (selectable progression) | **(v3 최종) AI 인프라가 큰 우산. 그 안에 두 측면 — 전력 공급 vs 시스템. 처음엔 전력 공급 쪽 (→ AI 데이터센터 전력 → SMR → 원전 시뮬레이션 + 자율운전 + 세 전공) → 점점 시스템 쪽으로 이동 (→ 컴퓨터 시스템 관심)** | 단순 chronology가 아닌, AI 인프라라는 큰 frame 안에서 **두 측면을 오가는 narrative**. 청자가 'AI 인프라'라는 큰 우산을 한 번 듣고, 본인의 academic path가 그 안에서 어떻게 진화했는지 명확히 파악 가능. 학부 수업 → **대학원 수업**으로 정확히 변경 (NQE625는 대학원 수업) |
| **2단계 3-way 비교 추가** | (v3 초안) 원자력 vs 컴퓨터 시스템 2-way 비교 ("안전 규제 → 정년퇴직" vs "빠른 결과") | **(v3 최종) AI / 원자력 / 컴퓨터 시스템 3-way 비교 — 핵심 기준은 "한 논문 contribution에 필요한 배경지식의 양"** | 본인이 의식적으로 세 길을 비교한 후 컴퓨터 시스템을 선택했다는 narrative depth 추가. "막연히 끌려서"가 아니라 "분석 후 결정"이라는 인상. 청자(시스템 lab 사람들)에게 thinking process 보여줌 |
| **LLM 시대 경쟁력 직관** | (v3 초안) "AI는 빠른데 영역 좁음" 정도의 표면적 비교 | **"AI 논문은 한 논문 안에 모델 설계가 다 들어 있어서 진입이 빠른 대신 배경지식이 적음" + "컴퓨터 시스템은 여러 layer 누적 이해가 필요 → 배경지식 ↑ → LLM 탐색 비용 ↑ → 사람 직관이 오래 가치를 유지할 영역"** | 본인의 진짜 framing 반영. 단순 "영역이 넓다/좁다"가 아니라 **LLM 시대 사람 연구자의 경쟁력** 관점. 시스템 lab 청자들이 즉시 공감할 화두. AI 연구자를 깎아내리는 톤이 아니라 분야별 진입장벽의 trade-off를 정직하게 인지 |
| **논문 영향력 지속성 추가** | (v3 중간) 배경지식 양만 비교 기준 | **두 번째 비교 축 — "그 논문의 영향력이 얼마나 오래 가는가"** 추가. AI는 NeRF→3DGS로 1~2년에 갈아탐 / 컴퓨터 시스템은 한 흐름이 더 오래 살아남아 한 연구를 깊이 이어갈 수 있음 | "배경지식 ↑ → LLM 탐색 비용 ↑"와 "영향력 오래 감 → 깊이 이어갈 수 있음"이 같은 방향으로 reinforce. **본인이 한 분야를 길게 파는 사람**이라는 character signal. NeRF→3DGS는 본인 CS479 과제 경험 (knowledge proximity OK) |
| **3-way 비교 톤 humble화** | (v3 강) "AI 연구는 ~ 빨리 바뀌어요" / "원자력 연구는 ~인데" / "컴퓨터 시스템은 ~ 필요해요" — 단정 톤 | **"제 나름대로 비교해봤어요. 아직 깊이 연구해본 게 아니라 표면적 인상에 가깝지만, 제 눈에 들어온 차이는…"** + 각 분야 묘사에 "느껴졌고 / 같았어요 / 보였어요 / ~이 아닐까 싶었고" 같은 주관적 qualifier 삽입 | 학부생이 세 분야를 단정 톤으로 평가하면 거만하게 들림. 교수님 앞 자기소개에서는 **"내가 그렇게 느꼈다, 나의 생각을 공유한다"**는 톤이 정직하고 적절. character 약화 X — 사고의 깊이는 보존되고, 표현만 humble |
| **전반적 공손 톤 통일 (~요 → ~습니다)** | 본문 곳곳에 "~어요 / 거예요 / 거요 / 더라고요" 해요체 + ~습니다 합쇼체 혼재 | **전반적으로 ~습니다 합쇼체로 통일**: "들어왔어요 → 들어왔습니다" / "끌리더라고요 → 끌리게 됐습니다" / "사례예요 → 사례입니다" / "연구예요 → 연구입니다" / "~잖아요 → ~습니다" 등. ## ✅ 전달 체크리스트

- **분량**: 약 5분 35초 (클로징 learning goal 문장 제거로 추가 ~10초 ↓). 5분에서 35초 over. 더 줄이려면 ① 3단계 마개조 hook 한 줄 압축 (~5초) ② 클로징 메시지/전원 sub-problem list 압축 (~10초) ③ 1단계 세 전공 이유 약간 압축 (~10초) — 합치면 5분 10초 권. 식사 자리 자기소개로 5분 30초도 OK
- **위트 2개** (공손 톤으로 정리되며 wit 비중 축소됨):
  1. 정년퇴직 ("제가 만든 게 돌아가는 걸 보기도 전에")
  2. "메모리 용량 한계은 우회할 수 있는 것이구나" (감탄 톤)

- **톤**: 친구한테 설명하듯이. 단어가 어려워졌다 싶으면 바로 풀어 씀
  - "그게 X예요" → "**X라는 게**, ~한 거예요" 패턴 자주 사용
  - 예: "**SMR이라고**, 작은 원자로가" / "**TrainingCXL이라는 연구**, 그래픽카드가..."

- **세 곳에 한 박자 멈춤**:
  1. "저는 처음부터 **AI 인프라**에 기여하고 싶었습니다" **(0.5초)** "처음엔 전력 공급 쪽에, 나중엔..."
  2. "정년퇴직할 것 같다고 느꼈습니다" **(0.5초)** "반면 컴퓨터 시스템은..."
  3. "제가 가장 배우고 싶은 환경이라고 생각했습니다" **(0.5초)** "아직 배울 게..."

- **외워야 할 vs 즉석**:
  - **외워야 함**: 오프닝 첫 두 문장, 클로징 두 문장 (시작·끝 인상)
  - **3단계의 TrainingCXL/LightPC 두 문장**도 키워드 흐름 외울 것:
    - TrainingCXL = "그래픽카드가 본체 바깥의 큰 메모리까지 직접 갖다 쓰는 사례"
    - LightPC = "전원 꺼졌다 켜져도 작업 상태가 돌아오게 / 사용자가 손대지 않아도"
  - **즉석 가능**: 1·2단계, 4단계 본문 — 키워드만 잡고

- **국적 언급 금지**: 개조 그래픽카드 부분은 "누군가 그래픽카드를 뜯어서"로만

- **외래어 발음**: TrainingCXL, LightPC, CAMEL Lab, SMR, MARS, MATLAB — 또박또박. 한국어 흐름 중간에 살짝 호흡 두고 발음

## ❓ Follow-up 답변 카드

> v3 본문을 듣고 청자가 더 깊이 물어올 가능성 있는 5가지. 답변은 plain language로, 단 정확하게.

> [!tip]- Q1. "TrainingCXL에서 그래픽카드가 본체 바깥 메모리를 어떻게 직접 갖다 쓰나요? 그게 그렇게 단순하진 않을 텐데"
> "맞습니다. **CXL이라는 새 연결 규격**의 어떤 기능 덕분인 걸로 알고 있습니다. 그래픽카드와 그 바깥 메모리가 **같은 캐시 도메인**에 들어가서, 그래픽카드가 메모리를 자기 거처럼 쓰면서 일관성도 자동으로 유지된다는 부분이 핵심이라고 이해했습니다. cudaMemcpy 같은 명시적 복사가 없어지는 게 인상 깊었습니다."

> [!tip]- Q2. "LightPC가 전원 꺼졌다 켜져도 살아남는 거, 그런 컨셉이 처음은 아닐 텐데요"
> "네, 컨셉 자체는 이전 연구들이 있는 걸로 알고 있습니다. LightPC가 새로 한 건, **DRAM 없이 비휘발 메모리만으로** 시스템 전체를 그렇게 만들고, 그것도 **전원 꺼지기 직전 16ms 안에** 다 처리한 거였다고 이해하고 있습니다. 컨셉보다는 그걸 실제 silicon에서 검증한 그 방식이 인상 깊었습니다."

> [!tip]- Q3. "여러 대 컴퓨터로 학습 상태 잡아내는 거, 그게 LightPC의 단순 확장은 아니죠?"
> "네, 새로운 문제들이 따로 들어온다는 걸 알고 있습니다 — 컴퓨터들 사이를 오가는 메시지를 어떻게 같이 잡아낼지, 한 대만 전원 차단되면 어떻게 처리할지 같은 것들이요. 단순 확장이 아니라, LightPC의 **사상**을 어떻게 분산 환경에 보존할 수 있는지가 진짜 어려운 부분이라고 봅니다. 그래서 더 도전해보고 싶었습니다."

> [!tip]- Q4. "AI 학습 시스템을 직접 만져봤다고 했는데, 어떤 경험인가요?"
> *(본인 경험으로 채우기 — 예: Point Transformer, NeRF, 3DGS, 학부 연구 등)*

> [!tip]- Q5. "transparent라는 말은 어떤 의미로 쓰신 거예요?"
> "사용자가 학습 코드에 따로 손대지 않아도, 시스템이 알아서 상태를 살려준다는 의미로 썼습니다. PyTorch에서 명시적으로 checkpoint를 저장하는 방식이 아니라, **하드웨어나 OS layer에서 자동으로** 일어나게 하는 방향이요."
